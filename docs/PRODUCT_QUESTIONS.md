# Product Questions

## Confirmed Decisions

1. 품의 엑셀 양식 업로드는 기본 흐름에서 사용하지 않는다.
2. 견적서는 PDF가 중심이고 여러 장일 수 있다. 이미지 파일도 같이 허용한다.
3. 품의서 내부 필드는 사용자가 올리는 엑셀 양식을 참고한다.
4. Gemini API 키는 앱에서 직접 입력하고 한 번 입력하면 로컬에 저장한다.
5. 추출 결과는 사용자가 확인/수정한 뒤 Excel 생성으로 넘어간다.
6. 결과 파일명은 견적서 원본 파일명을 따른다.
7. PDF가 여러 개면 하나의 Excel 파일로 만든다.
8. 엑셀 저장 컬럼은 `내용`, `규격`, `수량`, `단위`, `예상단가`, `예상금액`이다.
9. API 키 저장은 일반 로컬 설정 파일 방식으로 충분하다.

## Highest Priority Open Questions

1. 여러 PDF가 서로 다른 업체 견적서일 때도 하나의 품의서로 합칠까요, 아니면 나중에 비교견적 기능으로 분리할까요?
2. 품목별 단위가 비어 있을 때 기본값을 `개`로 넣을까요, 빈칸으로 둘까요?
3. 업체명/견적일/합계금액 같은 요약 정보도 별도 시트에 남겨야 할까요?

## Extraction Details

1. 물품 행에서 꼭 필요한 열은 무엇인가요? 예: 품명, 규격, 수량, 단위, 단가, 공급가액, 세액, 합계.
2. VAT가 없는 견적서나 총액만 있는 견적서도 처리해야 하나요?
3. 여러 업체 견적서를 넣고 비교표까지 만들 계획이 있나요?
4. 수기로 수정한 내용은 다음 작업에도 기본값으로 저장해야 하나요?

## Workflow

1. 사용자가 추출 결과를 확인하고 수정하는 단계는 반드시 필요하다고 보면 될까요?
2. Excel 생성 후 자동으로 파일을 열어주는 기능이 필요할까요?
3. 저장 위치는 매번 선택하게 할까요, 아니면 기본 폴더에 자동 저장할까요?
4. 파일명 규칙이 있나요? 예: `YYYYMMDD_업체명_품의서.xlsx`.

## Deployment

1. Windows 배포는 설치형 프로그램이 좋나요, 아니면 압축을 풀고 실행하는 portable exe가 좋나요?
2. 학교 PC에서 인터넷 접속 또는 Gemini API 접속이 제한될 가능성이 있나요?
3. 프로그램 사용자는 한 명인가요, 여러 선생님이 각자 PC에서 사용할 예정인가요?

## Future Features

1. 나중에 추가하고 싶은 기능은 어떤 순서인가요? 예: PDF 지원, 다중 견적 비교, NEIS/K-에듀파인 양식 연동, 거래처 목록 저장.
2. 개인정보 또는 학교 내부 정보가 포함될 수 있으니 로컬 로그 저장을 최소화해야 할까요?
3. OCR/AI 결과를 누적해서 더 편하게 자동완성하는 기능이 필요할까요?
